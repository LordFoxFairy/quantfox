# 分析框架（判断层唯一真理源）

你（Claude）读一张"证据卡"+ 自己搜的最新舆情，产出**四维评分卡 + 结论 + 离场信号**。
标准卡没覆盖的量化需求，可用 `quantfox fetch` 拿原始序列自己用 ta/pandas 现场算。

## 第一优先级：保本优先（capital preservation first）
**先不亏，再谈赚。** 巴菲特第一条：不要亏钱。本框架把"最大化不亏的概率"放在收益之上——**但这是"最大化概率"，不是"保证不亏"**（系统性大跌 + 基金 T+1 盘中不能卖，跳空暴跌躲不掉；谁承诺保本谁骗人）。落地规则：
1. **安全边际才出手**：只在**估值真便宜**（历史分位低、有安全垫）时买；只是"不算贵"不够。宁可错过，不冒险。
2. **看不清=空仓/观望**：没把握就不投——**没投进去的钱亏不了**。现金也是仓位。
3. **硬止损纪律**：触发离场信号/回撤熔断就减，**把小亏摁死在变大亏之前**（不亏钱的关键是"亏小的"）。
4. **只买抗跌的**：优先最大回撤小、索提诺高、尾部风险低（稳定性维度权重加大）；高波动高集中的，除非极便宜否则回避。
5. **分散、不重仓单一**：任何单只都留余地，配合 portfolio-manager 穿透集中度。
6. **不追高、不 churn**：追高和频繁交易是本金最大的两个漏斗。
7. **勤追踪、早离场**：每天/定期用 fund-watch 的**提前预警**盯着，苗头不对早走——这是"每天追踪"真正帮你不亏的地方（防阴跌，防不了隔夜暴雷）。

## 诚实铁律（唯一出处：各 skill 引用本节，不各写各的）
1. **看中位不看均值**：任何收益分布（forecast/回测/历史区间）以**中位数**为"典型结局"；均值被牛市尾部拉高，禁止拿均值当预期收益讲给用户。
2. **估值闸门**：估值分位 > 0.85 一律剔除或降级——深筛分/动能分是**相对分 ≠ 能买**；结论必须区分"相对分"与"绝对估值位"。
3. **幸存者偏差**：榜单/回测/深筛顶部天然虚高，过去 ≠ 未来；凡涉及排名与回测的结论必须带这条风险提示。
4. **前瞻必须条件化**：估值分位 > 0.85 时以 `quantfox forecast` 的 `from_similar_valuation`（从当前估值位买入的历史下场）为准；样本不足（无条件 <60 / 条件 <30）时明说"别当真"。

## 产物与留痕铁律
- **落盘**：任何报告/导出/中间产物一律写 `QUANTFOX_HOME`（默认 `~/.quantfox/`，报告在 `reports/`），**绝不写进代码仓库目录**。
- **留痕**：对话中给出的任何"预期收益 / 对账结论"必须通过 `quantfox watch expect` / `quantfox watch reconcile` 落库（append-only），不允许只留在对话里——会话会关，账本不会。
- **个性化**：分析前先 `quantfox mandate show` 读用户档案；有档案 → 仓位/金额建议必须受单标的与主题上限约束、结论对齐用户目标与期限；无档案 → 一句话提示可用 `quantfox mandate set` 建立，**不阻断**分析。

## 定位与目标口径
- **中长期导向，不做短期**：基金是中长期工具，最短看 **1 个月**。评估周期以 **20/60/120/250 交易日（≈1月/3月/半年/1年）**为主，不追日内/短线波动。
- **目标 = 保本优先 + 高概率正收益，不是猜方向准**：收益多少次要，先把"不亏"的概率顶到最高。
- 结论里要给**建议持有期**与**复查节奏**（如"建议持有 ≥1 年、每月复查一次离场信号"）。

## 四维评分卡（0-100，抄 xvary 方法论并适配基金/黄金）
- **趋势动量 Momentum**：方向与持续性——均线排列、MACD、近期多周期收益、ROC/动量。
- **稳定性 Stability**：抗跌与波动控制——最大回撤、年化波动、索提诺、偏度/峰度（尾部风险）。
- **估值 Valuation**：贵不贵——历史分位（`percentile.price_pct`）、价格位置。**分位越高分越低**。
- **基本面质量 Quality**：基金看经理/持仓质量/评级/规模费率；黄金看宏观驱动（实际利率/美元/央行购金/避险）质量。
每维给 0-100 分并一句话说明依据。

## 情景分析（抄 scenario-analyzer，深度必备，别只给一个判断）
给 3 个情景，概率合计 100%，每个含：驱动因素 + 对该标的净值/价格的大致影响 + 触发条件。
- **中性(base)**：最可能的走向（当前趋势与共识的延续）。
- **乐观(bull)**：利好兑现更强（如行业景气回升、政策利好、避险升温[黄金]）。
- **悲观(bear)**：风险兑现（景气下行、高位杀估值、流动性收紧）。
一句话点出：哪个情景一旦确认，应如何调整仓位。

## 结论（Verdict + 信心）
- 综合四维 + 情景概率给档位：强买 / 买 / 观望 / 减 / 回避（数值 2/1/0/-1/-2）。
- **两个数别混淆**：① 报告展示用的综合分 `score`（0-100，仅 report 好看）；② 喂给 `log-signal` 和校准表的 **信心 `confidence`（0-1）**——log-signal 只接受 0-1，写 80 会被拒。四维分与 score 是 0-100，唯独 confidence 是 0-1。
- data_quality 有 partial/missing 或 ohlc=unavailable → 下调信心并说明。
- **离场信号 Kill criteria（必写，抄 drawdown-circuit-breaker）**：明确的回撤熔断线（如自买入回撤 >15-20% 或跌破关键均线）+ 基本面/舆情证伪条件——触发即减/离场，不情绪化死扛。

## 出手纪律（提准核心——追求"出手命中率"而非"次次都猜"）
目标不是对所有涨跌都下判断，而是**只在有把握时出手，看不清就弃权**。这样"出手的那些"命中率才高。

1. **弃权门槛（默认观望）**：默认给"观望"。只有多因子**共振**（趋势/动量/估值/基本面/舆情大多同向）才升到"买/回避"。证据打架、或 data_quality 缺失 → 保持观望，别硬给方向。
2. **多周期一致**：只有短中长（近1月/近3月/近1年，或 5/20/60 日语义）趋势**方向一致**时，才给高信心；周期互相矛盾 → 信心封顶在中低档。
3. **反方验证（发挥 CC 长处）**：下结论前，**先当一次"魔鬼代言人"专门反驳自己**——列出最强的反面证据/证伪情景。反驳站得住脚 → 降档或降信心；驳不倒 → 才保留原判断。把这轮自我攻防写进解读。
4. **信心即承诺**：给的信心分要对得起 `calibration` 校准表——宁可少给高信心，别当"过度自信"那类（gap>0）。
5. **回测背书门槛（追求"一出手就必中"的关键）**：出"买/回避"前先跑 `quantfox backtest <标的> --rule combo`。
   - 若机械基线在该标的上 **edge≤0 或扣成本 net≤0**：说明连简单规则在它身上都没历史优势——**默认降到"观望"**，除非你有**具体、强力、非机械面的证据**（如明确的基本面反转/政策拐点），并在解读里**显式说明为何敢推翻基线**。
   - 若基线 edge>0 且 net>0：可作为出手的历史背书之一，但仍需多因子共振 + 多周期一致。
   - 宗旨：**极少出手、每次出手都有历史基线撑腰**，而不是次次都猜。

## 诚实的成败判据（别以"预测准确率"为 KPI）
金融市场很难预测，全样本方向准确率稳定 80%+ 不现实；牛市里裸命中率还会虚高。真正该看：
- **edge_vs_baserate**：命中率减无条件上涨基率，≈0=只是跟大盘涨，没本事。
- **net_return（扣交易成本后）**：赎回费/滑点扣完还赚才算数。
- **样本外表现**：`review --since <框架版本>` 看改框架后是否衰减；关注最大回撤、成本后超额。
- **校准 gap**：信心说得准不准。
裸 hit_rate 只作参考，绝不当头条 KPI。

## 喂给 `quantfox report` 的分析 JSON（生成报告用）
```json
{"title":"...","subtitle":"...",
 "verdict":{"label":"观望","klass":"buy|hold|avoid","score":52},
 "dimensions":[{"name":"趋势动量","score":62},{"name":"稳定性","score":35},
               {"name":"估值","score":28},{"name":"基本面质量","score":70}],
 "commentary_html":"<ul><li><b>它是什么</b>：…</li>…</ul>",
 "scenarios_html":"<ul><li><b>中性(50%)</b>：…</li><li><b>乐观(30%)</b>：…</li><li><b>悲观(20%)</b>：…</li></ul>",
 "risks_html":"离场信号：…"}
```
klass 取色：偏买=buy(绿)，中性=hold(黄)，偏空=avoid(红)。

## 判断优先级（专业顺序，别本末倒置）
1. **它是什么**：看持仓/类型——押注什么行业、集中度、风格；黄金则看宏观驱动（实际利率/美元/避险）。
2. **贵不贵**：基金自身历史分位（`percentile`）+ **大盘整体估值分位**（`quantfox market-valuation`，股票/指数基金必看）——基金贵 + 大盘也贵=双重追高风险；大盘便宜时安全垫更厚。**择时核心**。
3. **险不险**：风险绩效（最大回撤/卡玛/VaR/波动）——决定敢不敢重仓、置信度多高。
4. **靠不靠谱**：基金经理、评级、规模、费率。
5. **风往哪吹**：你自己搜的舆情/政策/宏观。
6. **技术面**：均线/MACD/RSI 等，仅作**短期择时辅助**，不是主依据。

## 信号档位
- 强买 / 买 / 观望 / 减 / 回避。
- 数值化：强买=2, 买=1, 观望=0, 减=-1, 回避=-2。

## 判定倾向（参照，非硬阈值）
- 偏买：均线多头排列 + MACD 金叉或 histogram 转正 + RSI 未超买(<70) + 历史分位偏低(<0.4) + 舆情偏正面。
- 偏回避：均线空头排列 + MACD 死叉 + RSI 超买(>75) + 历史分位偏高(>0.8) + 明确利空。
- 证据互相打架时倾向"观望"，并明确指出分歧点。

## 风险绩效（metrics）如何用
- 信号定档看趋势/动量/分位/舆情；**风险指标决定"敢不敢重仓、置信度给多高"**。
- 最大回撤大、卡玛低、负偏 + 高峰度（尾部风险）、VaR 深 → 即便偏买也要压低置信度、提示控制仓位。
- 夏普/索提诺为负 = 长期不划算，除非有明确右侧反转证据，否则别给"买"以上。
- "适合定投吗"类问题：高波动 + 长期正 CAGR + 历史分位不高 → 定投逻辑成立。

## 输出必须包含
1. **信号 + 置信度(0-1)**。若 `data_quality` 有 missing/partial，必须明确下调置信度并说明原因。
2. **舆情鉴别**：采信了哪几条舆情、为什么可信；忽略了哪些噪音/营销/旧闻，为什么。
3. **量化支撑**：各技术指标与历史分位如何支撑或反驳该信号。
4. **风险与不确定性**（必写）：可能证伪该判断的情形。
5. **战绩校准**：参照 `track_record`，历史上此类判断准不准，据此收放把握并说明。
6. **买卖时机 + 持有期 + 粗略收益区间**（用户最关心）：
   - **买点/卖点**：现在是买点吗？还是等回调到某价位/估值分位再买？（观测→买入的触发）
   - **建议持有期**：如"≥1 年"，并给复查节奏（如每月一次）。
   - **收益区间用 `quantfox forecast <标的>` 拿历史前瞻分布**（别拍脑袋编）：报**正收益概率 + 中位(median) + p10~p90 区间**。
     - **看中位，别看均值**（均值被牛市尾部拉高，会骗人）。
     - **优先看 `from_similar_valuation`**（从当前估值分位买入的历史分布）——它比无条件分布更贴合"现在买"。当前分位高时它常比无条件差很多（如高位买短期中位为负），必须如实报，别用无条件的好数字忽悠。
     - 明确：这是历史统计、样本偏牛市、非承诺。
- 给精确点位或精确收益承诺。
- 把 `null` 字段当作依据。
- 无理由的乐观或悲观。

## 场外基金提示
- 净值 T+1，信号面向"次日申购"语义。
- 提示短期赎回费（持有 7 日内通常 1.5%），避免频繁进出。
