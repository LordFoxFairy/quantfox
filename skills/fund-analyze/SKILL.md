# 基金 / 黄金分析 SOP

这是一份**标准作业流程（SOP）**。真正的判断力是你（CC agent）本身，这份 SOP 负责把流程、数据、判断标准组织好，让你稳定地给出专业、可解释、可复盘的结论。

## 心法：先懂分工

| 角色 | 职责 | 原则 |
|---|---|---|
| `money` 引擎 | 取**专业数据**（基金经理/持仓/评级/规模费率 + 净值/OHLC）+ **预计算风险绩效与指标** + **存档复盘** | 这些是你自己拿不到的护城河；要进复盘、要可比、要可靠的用预计算 |
| **你（agent）** | **舆情/宏观自己搜自己判**（WebSearch）+ 综合判断 + 适配用户真实问题 + 临场补算 | 舆情用 WebSearch 比任何固定数据源都新都准；标准卡没覆盖的用 `money fetch` 自己算 |

一句话：**引擎给专业数据 + 可复盘的预计算；判断、舆情、临场探索是你的活。**

**重要认知**：技术指标（MA/MACD/KDJ）只是**辅助**。真正专业的判断靠：基金**持仓看它是什么**、**历史估值分位看贵不贵**、**风险绩效看险不险**、**基金经理看靠不靠谱**、**舆情宏观看风往哪吹**。

## 前置：引擎就绪（首次）

```
bash skills/fund-analyze/scripts/setup.sh
```

## 命令速查（引擎 = 数据 + 预计算）

| 命令 | 给你什么 |
|---|---|
| `uv run money evidence <标的> --format json` | **主命令**：完整证据卡（基本面/风险绩效/估值分位/指标/战绩/数据质量） |
| `uv run money profile <标的>` | 基金基本面：经理/持仓/评级（黄金不适用） |
| `uv run money metrics <标的>` | 风险绩效（夏普/索提诺/卡玛/VaR/回撤…） |
| `uv run money indicators <标的>` | 技术指标（辅助） |
| `uv run money fetch <标的>` | 原始净值/价格序列（**你要自己算额外东西时用这个**） |
| `uv run money review <标的>` / `--all` | 历史战绩 |
| `uv run money log-signal ...` | 存档本次预测 |

字段怎么读 → 先看 `references/evidence-card.md`（第一次分析前必读）。
判断标准 → 严格遵守 `money/prompts/analysis_framework.md`。

## 主流程

1. **确定标的**：提取基金 6 位代码或识别"黄金"；含糊先问清。
2. **看战绩、校准自己**：`uv run money review <标的>`。历史上某类判断准就自信些，不准就收敛。
3. **取证据卡**：`uv run money evidence <标的> --format json`。
4. **专业地读它**（不是只看技术指标）：
   - **持仓**：前十大重仓是什么行业？集中度多高？→ 判断"这基金到底押注什么、风格漂没漂移"。
   - **历史估值分位** `percentile.price_pct`：接近 1=历史高位（贵，追高谨慎），接近 0=低位。**这是择时的核心**。
   - **风险绩效** `metrics`：最大回撤/卡玛/VaR → 判断"敢不敢重仓、最坏亏多少"。
   - **基金经理/评级/规模费率**：经理是谁、评级几星、规模是否过大、费率是否偏高。
   - 技术指标 `indicators` 仅作短期择时的**辅助**参考。
5. **查数据质量**：`data_quality` 有 `partial`/`missing`、或 `ohlc=unavailable`（基金无 ATR/KDJ 等）→ **明说并下调置信度**，绝不拿 null 硬解读。
6. **舆情/宏观自己搜自己判**：用 WebSearch / WebFetch 搜该标的（或其重仓行业、或黄金对应的实际利率/美元/央行购金）的最新新闻、政策、研报。鉴别：哪条可信、哪条是营销/旧闻、利好还是利空、情绪多强。**这一步是你 CC 的主场，别依赖固定数据源。**
7. **按框架推理**：读 `money/prompts/analysis_framework.md`，综合基本面 + 估值 + 风险 + 舆情 + 战绩下判断。
8. **需要标准卡没有的？自己算**：比如想要 30 日滚动夏普、自定义均线、和某指数的相关性——`uv run money fetch <标的>` 拿序列，用 `ta`/pandas 现场算，不必改引擎。
9. **产出结论**（见下方模板）。
10. **存档预测**（复盘的关键，别漏）：
   ```
   uv run money log-signal --symbol <代码> --type <otc_fund|gold> \
     --signal <档位> --signal-numeric <2..-2> --confidence <0-1> \
     --price-ref <证据卡最新价> --ts <今天YYYY-MM-DD> --horizons 5,20,60 \
     --rationale "<一句话理由>"
   ```

## 情形分派（这个 skill 不止会"分析一只"）

先判断用户到底在问什么，再侧重不同证据：

- **"能不能买 / 买点"** → 主流程，重点落在信号与择时（趋势 + 动量 + 历史分位 + 舆情）。
- **"风险多大"** → 重点讲 `metrics`：最大回撤、年化波动、VaR95、卡玛，讲清"最坏可能亏多少、回本要多久"。
- **"适合定投吗"** → 看长期趋势、波动、历史分位；讲定投的摊薄逻辑，而不是精确择时。高波动 + 长期向上的标的更适合定投。
- **"A 和 B 哪个好"** → 对每只各跑一次 `evidence`，横向对比夏普/最大回撤/趋势排列/历史分位/舆情，给出相对结论。
- **"最近怎么样"** → 简报式：一句趋势定调 + 3~4 个关键指标 + 舆情要点，不必长篇。

## 输出模板

```
【信号】强买/买/观望/减/回避（置信度 X%）

一、结论与理由（2-3 句，先说人话）
二、量化证据：趋势(均线/MACD) · 动量(RSI/KDJ) · 风险绩效(夏普/最大回撤/VaR) 各挑最关键的讲
三、舆情研判：采信了哪几条、为什么；忽略了哪些噪音、为什么
四、风险与不确定性：什么情况会证伪这个判断
五、战绩校准：历史上这类判断准不准，据此我收/放了多少把握

⚠️ 本分析基于公开数据，非投资建议，不保证盈利，决策与风险由你自行承担。
```

## 铁律

- **不承诺精确点位或收益**。基金/黄金接近随机游走，你给的是有依据的概率倾向，不是水晶球。
- `data_quality` 缺失 → 明说 + 下调置信度，绝不拿 `null` 当依据。
- 基金只有净值 → OHLC 类指标不可用，别硬解读；黄金才有。
- 结尾必附免责，最终决策与风险由用户承担。
