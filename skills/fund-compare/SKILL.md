# fund-compare — 多标的横向对比（闭环）

对每只标的取同一套证据，按统一维度横向比，给相对结论——不是各自孤立分析，而是"放一起谁更优"。

## 诚实铁律 + 档案 + 留痕（出处：quantfox/prompts/analysis_framework.md v15，以框架为准）
- **看中位不看均值**；估值分位 **>0.85 过闸门**（剔除/降级，相对分≠能买）；涉及排名/回测必提**幸存者偏差**；高位前瞻以 `from_similar_valuation` 为准，样本不足明说别当真。
- **第 0 步先 `quantfox mandate show`**：有档案 → 金额/仓位建议受单标的与主题上限约束，结论对齐用户目标与期限；无档案 → 一句话提示可 `quantfox mandate set` 建立，不阻断。
- **产物落 `QUANTFOX_HOME`**（默认 `~/.quantfox/`），绝不写进代码仓库；对话里的预期收益/对账结论用 `quantfox watch expect` / `watch reconcile` 落库，不许只留在对话里。

## 闭环步骤
1. **列清标的**：从用户话里提取所有代码/"黄金"（≥2 个）。
2. **逐只取证据**：对每只 `quantfox evidence <标的> --format json`。
3. **统一维度横向比**（做成对比表）：
   - **趋势/动量**：均线排列、近1年/近3月收益。
   - **风险**：最大回撤、年化波动、夏普、卡玛——**风险调整后收益是对比核心**。
   - **估值**：历史分位（谁更便宜/更贵）。
   - **基本面**（基金）：经理、持仓行业、评级、规模费率；同类型才好直接比。
   - **前瞻分布对比**：逐只跑 `quantfox forecast <代码>`，对比中位（不是均值）与正收益概率；估值分位 >0.85 的用 `from_similar_valuation` 口径对比，别拿山顶上的无条件数字比山脚的。
4. **舆情增量**：必要时 WebSearch 补各自最新动态。
5. **相对排序与结论**：给排序 + 各自"更适合谁/什么场景"（如"A 进攻性强但回撤大适合能扛的；B 稳健适合保守"），别只说"A 好"。
6.（可选）**生成对比报告**：对主推的 1-2 只用 fund-analyze 出可视化报告。

## 铁律
- 只做相对比较，不承诺绝对涨跌。
- 苹果比苹果：不同类型基金（如股票型 vs 债券型）要点明不可简单同比。
- 结尾必附免责，决策与风险由用户承担。
