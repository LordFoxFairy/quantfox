# fund-watch — 持仓监控（opt-in · 触发式 · 中长期少动）

**理念（务必守）**：普通用户不想被噪音轰炸。这套只做三件事——① 只看用户**自己加进来**的持仓；② 平时**沉默**，只有**真触发**才提醒；③ 中长期导向，**默认继续持有，极少建议动作**，坚决不 churn。

## 前提
- 监控清单是**用户主动维护**的：`uv run quantfox watch add <代码> <买入价> <买入日期>`（黄金同理）。
- 清单为空 → 不要臆造，直接告诉用户"先把你持有的基金加进 watch 清单我才能盯"。

## 闭环步骤
1. **快扫**：`uv run quantfox watch check` → 每只持仓的客观信号（浮盈亏、自买入回撤、是否跌破 MA60、估值分位）+ `need_attention` 列表。
2. **没触发**：直接回"✅ 持仓一切正常，继续持有，无需动作"，附一句各只的浮盈亏。**到此结束，别过度分析。**
3. **有触发**：只对 `need_attention` 里的标的深看——`uv run quantfox evidence <代码>` + WebSearch 查是不是有实质利空/基本面变化，判断触发是"该真离场"还是"正常波动别慌"。
4. **给动作建议**（仅触发的）：减仓/离场/继续持有各自理由；结合 position-sizer 的熔断纪律。中长期下，**除非触发离场信号或基本面恶化，一律建议拿住**。
5. **诚实结尾**：非投资建议；提示"频繁看盘、追涨杀跌是中长期最大的敌人"。

## 关于定时（是用户的选择，我不擅自建）
- 想自动定时：建议用 Claude Code 的 `/schedule` 挂一个**周频**（中长期不必日频）任务调用本技能；具体节奏由用户定。
- 不想定时：随时说"复查下我的持仓"即可按需触发。
- **绝不**替用户偷偷创建定时任务。

## 铁律
- 只监控用户 opt-in 的标的；清单空就不编。
- 平时沉默，只在真触发时出声；中长期少动、不 churn。
- 结尾必附免责，决策与风险由用户承担。
