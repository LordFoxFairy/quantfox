# portfolio-manager — 基金组合体检与再平衡（闭环，适配自上游同名 skill）

分析"你已经持有的一篮子"，不是孤立看每只，而是看**整体是否合理**。核心适配点：**基金要穿透到底层持仓看真实集中度**——两只"不同"的基金可能都重仓半导体，看似分散实则押注单一赛道。

## 诚实铁律 + 档案 + 留痕（出处：quantfox/prompts/analysis_framework.md v16，以框架为准）
- **看中位不看均值**；估值分位 **>0.85 过闸门**（剔除/降级，相对分≠能买）；涉及排名/回测必提**幸存者偏差**；高位前瞻以 `from_similar_valuation` 为准，样本不足明说别当真。
- **第 0 步先 `quantfox mandate show`**：有档案 → 金额/仓位建议受单标的与主题上限约束，结论对齐用户目标与期限；无档案 → 一句话提示可 `quantfox mandate set` 建立，不阻断。
- **产物落 `QUANTFOX_HOME`**（默认 `~/.quantfox/`），绝不写进代码仓库；对话里的预期收益/对账结论用 `quantfox watch expect` / `watch reconcile` 落库，不许只留在对话里。

## 需要用户给的
- 各持仓：基金代码 + 金额或占比（可含黄金）。缺就先问。风险偏好可选。

## 闭环步骤（抄 portfolio-manager 五维）
1. **逐只取证据**：对每只 `quantfox evidence <代码> --format json`（拿持仓/类型/风险/估值）。
2. **配置结构**：按基金类型（股票/混合/债/QDII/黄金）、风格汇总占比，对照用户风险偏好看是否失衡。
3. **穿透集中度（关键）**：把各基金的前十大重仓**合并穿透**，看：
   - 单一个股跨基金合计权重（>10-15% 预警）；
   - 单一行业/赛道跨基金合计权重（>30-40% 预警）；
   - 高度重叠的基金（持仓雷同）视为**冗余**，起不到分散作用。
4. **组合风险**：加权最大回撤/波动；是否过度集中在高波动标的。
5. **逐持仓前瞻汇总**：对每只持仓跑 `quantfox forecast <代码>`，汇总"组合里每只从当前位置的中位赔率"；高位（分位>0.85）持仓用 `from_similar_valuation`。全组合中位赔率恶化是减仓信号之一。
6. **逐只裁决**：每只给 **留 / 加 / 减 / 换**，说明理由（估值贵了、业绩掉队、与其它重复、风格漂移等）。
7. **再平衡建议**：指出超配/低配，按紧急度排序，给出调仓方向与大致金额；提示分批、避免情绪化一次性大调。
8. **输出组合体检报告**：整体结论 + 配置表 + 穿透集中度预警 + 逐只裁决 + 再平衡清单 + 免责。

## 阈值参考（抄 portfolio-manager 并适配基金）
- 单一穿透个股 >10-15%、单一行业 >30-40% → 预警。
- 基金只数：太少(<3)欠分散、太多(>15)过度分散且难管理。
- 持仓重叠度高的两只 → 建议合并留一。

## 铁律
- 必须穿透到底层持仓看真实集中度，别被"买了多只"的表象骗。
- 只给结构性建议，不承诺涨跌；调仓提示分批、控制成本（赎回费）。
- 结尾必附免责，决策与风险由用户承担。
