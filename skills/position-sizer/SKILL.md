# position-sizer — 仓位与定投建议（闭环，适配自上游同名 skill）

**核心原则（抄 position-sizer）：生存优先——仓位管理是为了熬过连续亏损、活到反转，不是为了单次利润最大化。**

场外基金没有股票那样的硬止损价，所以把"单笔风险"换算成**用波动率/最大回撤估计的下行风险预算**。

## 诚实铁律 + 档案 + 留痕（出处：quantfox/prompts/analysis_framework.md v14，以框架为准）
- **看中位不看均值**；估值分位 **>0.85 过闸门**（剔除/降级，相对分≠能买）；涉及排名/回测必提**幸存者偏差**；高位前瞻以 `from_similar_valuation` 为准，样本不足明说别当真。
- **第 0 步先 `quantfox mandate show`**：有档案 → 金额/仓位建议受单标的与主题上限约束，结论对齐用户目标与期限；无档案 → 一句话提示可 `quantfox mandate set` 建立，不阻断。
- **产物落 `QUANTFOX_HOME`**（默认 `~/.quantfox/`），绝不写进代码仓库；对话里的预期收益/对账结论用 `quantfox watch expect` / `watch reconcile` 落库，不许只留在对话里。

## 需要用户给的
- 可投入金额（或占总资产比例）、风险偏好（保守/稳健/进取）、已有持仓（可选）。缺就先问。

## 闭环步骤
1. **取风险数据**：`quantfox evidence <标的> --format json`，读 `metrics.ann_vol`、`metrics.max_drawdown`、`metrics.var95`、`percentile.price_pct`。
2. **定风险预算**（生存优先）：
   - 单一标的建议上限：保守 ≤10%、稳健 ≤20%、进取 ≤30% 的总资产；高波动（年化波动 >30%）或高分位（>0.8）再打 7 折。
   - 用最大回撤做压力测试：算"若重演历史最大回撤，这笔会亏多少钱"，让用户能承受再上。
3. **一次性 vs 定投**：
   - 高波动 / 高估值分位 / 长期向上 → **倾向定投（分批摊薄）**；低分位 + 明确右侧 → 可较大比例一次性。
   - 定投给**期数与节奏**（如分 6-12 期、按周/月），每期金额 = 目标仓位 / 期数。
4. **加减仓与离场纪律**：
   - 分位越高越要分批、越谨慎；跌破关键均线或触发 fund-analyze 的离场信号（Kill criteria）→ 减仓。
   - 明确"什么情况加、什么情况减、什么情况清"。
5. **输出建议**：目标仓位金额与占比 + 一次性/定投方案（期数×每期金额）+ 压力测试（最坏亏多少）+ 加减仓规则 + 免责。

## 铁律
- 不替用户决定方向；这里只管"投多少、怎么分批、何时加减"。
- 任何单一标的都要留有余地，别 all-in；高波动高估值必须分批。
- 结尾必附免责，决策与风险由用户承担。
