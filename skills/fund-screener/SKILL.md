# fund-screener — 全市场选基（两级漏斗，稳字当头）

从上万只里选"最好"，**绝不能直接按收益排个榜就交给用户**——那是"追涨杀跌"的散户陷阱：涨得最猛的往往也最贵、最危险。专业做法是**两级漏斗**。

## 核心纪律
- **粗筛的榜单是按历史收益排的，天然偏向"已经涨完的"**。它只是**候选池**，不是推荐。
- **必须精筛降温**：候选里高估值、高回撤、扎堆单一赛道的要降级或剔除。
- 过去收益 ≠ 未来收益，结论里必须说清。

## 闭环步骤

1. **明确需求**：类型（股票/混合/债/指数/QDII/FOF）、风险偏好、赛道/主题、投资期限。含糊先问。
2. **粗筛出候选池**：
   `quantfox screen --type <类型> --top 100`（要常青稳健加 `--consistent`：近1年&近3年都前25%）。
   得到按"长周期加权+一致性"打分的 Top-N（每只带 score、consistent、多周期收益）。
3. **精筛降温（关键，别跳过）**：对候选池前 10-20 只逐一
   `quantfox evidence <代码> --format json`，检查：
   - **估值分位**：`percentile.price_pct` 高（>0.85）= 已在高位，**降级**（别追高）。
   - **风险**：最大回撤过大、夏普过低 → 降级。
   - **持仓穿透**：若前几名全扎堆同一赛道（如全是半导体）→ 只留代表 1-2 只，**强制分散**，别给用户一篮子同质基金。
   - **基金经理/规模/费率**：经理稳定、规模适中、费率低者优先。
4. **舆情降温**：对领跑赛道 WebSearch 一下是否已过热/见顶信号。
5. **产出精选名单**：给最终 Top-N（通常比候选池短），每只一句话推荐理由 + **风险/估值提示**，并按风格/赛道分组避免同质。必要时对最优 1-3 只用 fund-analyze 出可视化报告深挖。
6. **诚实结尾**：说明"这是基于历史与当前证据的筛选，非保证；追高有风险；建议分批/定投（见 position-sizer）"。

## 铁律
- 稳字当头：宁可少推几只稳的，不堆一串涨疯了的。
- 高估值/高集中/追热的，必须显式提示风险或剔除。
- **幸存者偏差**：榜单只含仍存续的基金，清盘/合并的已消失，Top 全是幸存者、历史收益天然偏高——别把幸存者的过去当未来，结论里要点破。
- 结尾必附免责，决策与风险由用户承担。
