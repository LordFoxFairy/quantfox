# fund-screener — 全市场多因子深筛（三级漏斗，稳字当头）

从上万只里选"最好"，**绝不能直接把收益榜端给用户**——那样榜首必然是涨疯了的过热板块（追高陷阱）。`quantfox screen` 已升级为**多因子深筛**：赢家 + 动能不过热 + 回调不追高 + 去重分散。但它仍只是**候选池**，要再精筛 + 舆情。

## 核心纪律
- **`screen` 已内置多因子降温**（剔抛物线、罚超买、每主题≤2只），但 `overheated=true` 的仍要警惕，别选。
- **必须逐只精筛 + 舆情**：候选里高估值、RSI超买、买在52周高点、高回撤的要剔除。
- **按需选风格**（balanced/steady/pullback/momentum），别用一套筛所有人。
- 过去收益 ≠ 未来收益，结论里必须说清。

## 三级深筛流程（跑全市场，别只看前12）

1. **先明确需求（不问清就别筛）**：投资期限（短线1-3月 / 中长期）、风险偏好、想要的风格。据此选 `--style`：
   - `balanced`（默认，动能但不追山顶）· `steady`（更稳、更怕过热）· `pullback`（找回调买点）· `momentum`（认动能，仍标过热）。
2. **多因子粗筛全市场**：`quantfox screen --type <类型> --style <风格> --top 30 [--exclude-overheated]`。
   引擎已内置：**赢家(1/2/3年一致靠前) + 动能不过热(剔抛物线>55%) + 回调不追高 + A/C去重 + 每主题≤2只强制分散**。
   输出每只带 `theme`、`overheated`、`score`、多周期收益，还有 `theme_spread`（主题分布）。**`overheated=true` 的是山顶货，默认别选。**
3. **逐只精筛（在短名单上做真功课）**：对候选逐一 `quantfox evidence <代码> --format json` + `quantfox market-valuation`，看：
   - **估值分位** `percentile.price_pct`：>0.85 高位追高 → 降级/剔除；低分位有安全垫 → 加分。
   - **风险**：最大回撤、夏普、`ohlc.rsi/超买`；短线尤其怕买在 RSI 超买、52周高点。
   - **持仓穿透 + 经理/规模/费率**：看真实押注、别名义分散实则同赛道。
4. **舆情研判（配合深筛）**：对入围赛道/标的 WebSearch 最新舆情——板块是否高低切换、有没有见顶信号、政策利好利空。**这一步把量化筛出的短名单再过一遍现实。**
5. **产出精选名单**：按风格/赛道分组的 5-10 只，每只一句话理由 + 估值/风险/是否过热提示 + 适合谁。最优 1-3 只可用 fund-analyze 出可视化报告（`--pdf` 可邮件）。
6. **诚实结尾**：非推荐、不保证盈利；追高有风险；建议分批/定投（position-sizer）。

## 铁律
- **深筛分是"相对分"，≠ 现在能买**：它只说"幸存者里动能质量最好"，不代表现在买它安全（本会话实测 top 入围估值分位普遍 96-99%）。短名单**必须过 evidence 估值分位闸门**（`percentile.price_pct`>0.85 剔除/降级），结论里**区分"深筛分(相对) vs 估值位(绝对)"**。
- 稳字当头：宁可少推几只稳的，不堆一串涨疯了的。
- 高估值/高集中/追热的，必须显式提示风险或剔除。
- **幸存者偏差**：榜单只含仍存续的基金，清盘/合并的已消失，Top 全是幸存者、历史收益天然偏高——别把幸存者的过去当未来，结论里要点破。
- 结尾必附免责，决策与风险由用户承担。
