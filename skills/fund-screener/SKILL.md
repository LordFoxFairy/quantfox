# fund-screener — 全市场多因子深筛（三级漏斗，稳字当头）

从上万只里选"最好"，**绝不能直接把收益榜端给用户**——那样榜首必然是涨疯了的过热板块（追高陷阱）。`quantfox screen` 已升级为**多因子深筛**：赢家 + 动能不过热 + 回调不追高 + 去重分散。但它仍只是**候选池**，要再精筛 + 舆情。

## 诚实铁律 + 档案 + 留痕（出处：quantfox/prompts/analysis_framework.md v14，以框架为准）
- **看中位不看均值**；估值分位 **>0.85 过闸门**（剔除/降级，相对分≠能买）；涉及排名/回测必提**幸存者偏差**；高位前瞻以 `from_similar_valuation` 为准，样本不足明说别当真。
- **第 0 步先 `quantfox mandate show`**：有档案 → 金额/仓位建议受单标的与主题上限约束，结论对齐用户目标与期限；无档案 → 一句话提示可 `quantfox mandate set` 建立，不阻断。
- **产物落 `QUANTFOX_HOME`**（默认 `~/.quantfox/`），绝不写进代码仓库；对话里的预期收益/对账结论用 `quantfox watch expect` / `watch reconcile` 落库，不许只留在对话里。

## 核心纪律
- **`screen` 已内置多因子降温**（剔抛物线、罚超买、每主题≤2只），但 `overheated=true` 的仍要警惕，别选。
- **必须逐只精筛 + 舆情**：候选里高估值、RSI超买、买在52周高点、高回撤的要剔除。
- **按需选风格**（balanced/steady/pullback/momentum），别用一套筛所有人。
- 过去收益 ≠ 未来收益，结论里必须说清。

## 三级深筛流程（跑全市场，别只看前12）

1. **先看大盘 + 明确需求**：先 `quantfox market-valuation` 看全A大盘贵不贵——**偏贵/贵时建议 `--style steady` 或 `pullback`**，别追顶。再问清投资期限（短线1-3月/中长期）、风险偏好。风格：`balanced`（默认，动能不追山顶）·`steady`（更稳、更怕过热）·`pullback`（找回调买点）·`momentum`（认动能，仍标过热）。
2. **多因子粗筛全市场**：`quantfox screen --type <类型> --style <风格> --top 30 [--exclude-overheated]`。
   引擎内置：**赢家(1/2/3年一致靠前) + 动能不过热(剔抛物线>55%) + 回调不追高 + A/C去重 + 每主题≤2只强制分散**。输出带 `theme/overheated/score/多周期收益` + `theme_spread`。**`overheated=true` 是山顶货，默认别选。**
   - **要给用户一份初筛名单报告**：`quantfox screen-report --type <类型> --style <风格> --top 50 --pdf` → 生成含**大盘估值+主题分布+Top-k表(过热标红)**的 HTML+PDF。**发用户一律附 PDF**（邮箱看不了 HTML）。
3. **逐只精筛（在短名单上做真功课）**：对候选逐一 `quantfox evidence <代码> --format json` + `quantfox market-valuation`，看：
   - **估值分位** `percentile.price_pct`：>0.85 高位追高 → 降级/剔除；低分位有安全垫 → 加分。
   - **风险调整收益**（`metrics`）：看**卡玛(年化÷回撤)/夏普/索提诺**排序，不是看谁涨最多；找"同等回撤收益最高"的（有效前沿）。
   - **前瞻收益分布**：`quantfox forecast <代码>` 看持有1/3/6/12月的**正收益概率+中位**，**优先看 `from_similar_valuation`**（从当前估值买入的历史下场）；看中位不看均值。
   - **持仓穿透 + 归因**：看真实押注、别名义分散实则同赛道；分辨是经理真本事(跨行情稳、低回撤)还是单押热门赛道的 beta。
   - 完整思维见 `docs/quant-fund-selection-methodology.md`（10 条选基心智模型，每条一个检查动作）。
4. **舆情研判（配合深筛）**：对入围赛道/标的 WebSearch 最新舆情——板块是否高低切换、有没有见顶信号、政策利好利空。**这一步把量化筛出的短名单再过一遍现实。**
5. **产出精选名单**：按风格/赛道分组的 5-10 只，每只一句话理由 + 估值/风险/是否过热提示 + 适合谁。最优 1-3 只可用 fund-analyze 出可视化报告（`--pdf` 可邮件）。
6. **诚实结尾**：非推荐、不保证盈利；追高有风险；建议分批/定投（position-sizer）。

## 铁律
- **深筛分是"相对分"，≠ 现在能买**：它只说"幸存者里动能质量最好"，不代表现在买它安全（本会话实测 top 入围估值分位普遍 96-99%）。短名单**必须过 evidence 估值分位闸门**（`percentile.price_pct`>0.85 剔除/降级），结论里**区分"深筛分(相对) vs 估值位(绝对)"**。
- 稳字当头：宁可少推几只稳的，不堆一串涨疯了的。
- 高估值/高集中/追热的，必须显式提示风险或剔除。
- **幸存者偏差**：榜单只含仍存续的基金，清盘/合并的已消失，Top 全是幸存者、历史收益天然偏高——别把幸存者的过去当未来，结论里要点破。
- 结尾必附免责，决策与风险由用户承担。
